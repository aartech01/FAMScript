ID Naming Tip

        
Node IDs must be unique, contain no spaces. Recommended format: ROLE_GEN_NNN

        

          
Example IDMeaning

          
            
GROOM_G0_001Groom, generation 0

            
GROOM_DAD_P1_001Groom's father, generation +1

            
BRIDE_SIS_G0_002Bride's sister, generation 0

          
        

      


      
      


        
Basic Structure

        
graph TB
  ID["Label text"]
  A --> B
  A --- B


        
Node Definition

        
Define a node with a unique ID and a quoted label:

        
GROOM_G0_001["Arjun Kumar, 28 (M)"]
BRIDE_G0_001["Priya Sharma, 25 (F)"]

        
Put name, age, and gender indicator in the label — it appears inside the node box.


        
The Couple Junction Dot

        
Join a married couple through a small junction dot CP_NAME(( )), then point the dot at their children. This keeps every generation on one aligned row and prevents connector lines from crossing — no subgraph blocks needed.

        
DAD["Ramesh, 55 (M)"]
MOM["Sita, 52 (F)"]
SELF["Rohan, 28 (M)"]
SIS["Priya, 25 (F)"]
CP_DAD_MOM(( ))            %% couple junction dot

DAD ---|"father"| CP_DAD_MOM
MOM ---|"mother"| CP_DAD_MOM
CP_DAD_MOM -->|"son"| SELF
CP_DAD_MOM -->|"daughter"| SIS

        
Why a dot? Pointing two parents straight at two children makes a crossing pair of lines Dagre can't untangle. One shared dot removes the crossing and lines the generations up.


        
Edges (Relationships)

        
DAD ---|"father"| CP_DAD_MOM    %% spouse → couple dot (line)
CP_DAD_MOM -->|"son"| SELF      %% couple dot → child (arrow)
GDF -->|"father"| DAD           %% direct parent → child (arrow)
DAD -.->|"nephew"| FRIEND       %% step / cousin / nephew (dashed)

      


      
      

        
Edge Syntax Summary

        

          
Mermaid SyntaxVisualFamily Use

          
            
A -->|"label"| B→ solid arrowParent to child (blood / legal)

            
A ---|"label"| B— solid lineMarriage / partnership

            
A ==>|"label"| B⇒ bold arrowCore couple highlight

            
A -.->|"label"| B- -> dashedSibling, cousin, step, adoptive, godparent

Example — Just Nodes & Edges

        
graph TB
  DAD["Ramesh, 55 (M)"]
  MOM["Sita, 52 (F)"]
  SELF_G0_001["Rohan, 28 (M)"]
  CP_DAD_MOM(( ))

  DAD ---|"father"| CP_DAD_MOM
  MOM ---|"mother"| CP_DAD_MOM
  CP_DAD_MOM -->|"son"| SELF_G0_001

        
No colours in the code — SELF_G0_001 renders blue (it matches the ROLE_G0_NNN self pattern) and the rest render as white cards. Use the Mermaid Theme dropdown to switch the overall palette.

Wedding Combined Canvas

        
Wedding merges both family trees into one canvas with groom on the left and bride on the right:

        
    
          
  • The editor has Groom's Family and Bride's Family tabs — each holds its own Mermaid code.
    • 
          
  • On compile, both trees are automatically merged into a single diagram — groom on the left, bride on the right.
    • 
          
  • A marriage dot is added between the groom's and bride's G0 nodes, descending to a shared child centred between them.
    • 
          
  • The single canvas has zoom, pan, and fullscreen controls.
    • 
          
  • Export the combined tree as SVG, PNG, or Markdown.
    • 
        

        
Switch between Groom and Bride tabs freely — your text is saved automatically when you switch.


        
Adding a Custom Event (for Developers)

        
To add your own ceremony preset, add one entry to the EVENTS object in app.js:

        
myEvent: {
  name:    'My Ceremony',
  color:   '#7c3aed',               // accent hex colour
  ecLight: 'rgba(124,58,237,0.07)', // dark-bg tinted background
  dual:    false,                   // true = wedding dual canvas
  desc:    'Description shown in the event bar'
}

        
Then add a matching <option> in the <select id="event-select"> in index.html.
